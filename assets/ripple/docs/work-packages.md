## What This Means In Practice
- WPs are sequenced to move from co-design to validation to replication.
- WPs are also layered: social (WP1/WP3), technical (WP2/WP5), demonstration (WP6), packaging (WP4), scale-up (WP7), and control (WP8).
- If you track project execution by WPs, a practical critical path is:
- WP1/WP2/WP3/WP5 setup -> WP6 validation -> WP4 final toolbox -> WP7 uptake/replication, with WP8 continuously governing.
